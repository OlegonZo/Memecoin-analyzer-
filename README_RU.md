# Memecoin Analyzer — публичная объяснимая демоверсия

[![tests](https://github.com/OlegonZo/Memecoin-analyzer-/actions/workflows/tests.yml/badge.svg)](https://github.com/OlegonZo/Memecoin-analyzer-/actions/workflows/tests.yml)

[English version](README.md)

Небольшой запускаемый Python-проект для портфолио. Он показывает, как я
структурирую объяснимую оценку накопления токена кошельками: проверяю входной
снимок, отбрасываю пыль, учитываю вес наблюдений, считаю прозрачный балл и
применяю рыночные и риск-фильтры.

В репозитории используются **только синтетические токены и кошельки**. Здесь
нет реальных адресов, API-ключей, логов, приватных списков кошельков, исполнения
сделок или закрытой калибровки.

## Что показывает демоверсия

- типизированные неизменяемые модели `dataclass` и проверку входных данных;
- фильтрацию пыли и отрицательного потока до агрегации;
- взвешивание накопления с учётом качества наблюдаемого кошелька;
- проверки ликвидности, объёма, концентрации холдеров и полномочий токена;
- объяснимое решение `WATCH` / `REJECT` с результатом каждого фильтра;
- CLI без внешних зависимостей, детерминированные тесты и GitHub Actions.

`WATCH` означает только «изучить токен внимательнее». Это **не** торговый
сигнал, финансовый совет или команда на покупку.

## Как работает оценка

```text
синтетический JSON
        |
        v
проверка данных -> фильтр пыли -> веса кошельков -> балл 0–100
                                                        |
                                                        v
        ликвидность / объём / концентрация / полномочия токена
                                                        |
                                                        v
                                  объяснимое WATCH или REJECT
```

Публичный балл намеренно сделан простым:

- ширина накопления: до 30 баллов;
- объём покупок с учётом качества: до 50 баллов;
- среднее качество кошельков: до 20 баллов.

Все демонстрационные пороги находятся в `EvaluationPolicy`. Они нужны для
понятного примера и тестов и не являются рабочими параметрами приватной системы.

## Запуск

Нужен Python 3.11 или новее. Внешних зависимостей во время работы нет.

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
python -m pip install -e .

memecoin-analyzer-demo examples/synthetic_snapshot.json
```

Полный результат в JSON:

```bash
memecoin-analyzer-demo examples/synthetic_snapshot.json --json
```

Тесты:

```bash
python -m unittest discover -s tests -v
```

## Граница публичной версии

Мой рабочий исследовательский сканер — отдельная приватная система. В неё входят
сбор данных, исследование кошельков, откалиброванная логика обнаружения,
хранение истории, мониторинг и отправка уведомлений в Telegram. Эти компоненты,
реальные адреса и исторические наборы данных не публикуются из соображений
безопасности и защиты интеллектуальной собственности.

Этот код не является копией закрытого алгоритма. Это самостоятельная
sanitized-демоверсия инженерного подхода, который можно обсудить на собеседовании:
понятные границы, детерминированная оценка, объяснимость, тестирование и
безопасная работа с чувствительными данными.

## Ограничения

Мемкоины — высокорисковые активы. Проект не подключается к блокчейну или бирже,
не открывает сделки и не заявляет о доходности. Для реального применения нужны
отдельная проверка, мониторинг и риск-контроль.
